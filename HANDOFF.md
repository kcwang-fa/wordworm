# Word Worm 交接包——冒險模式二期工程

給 Vickie 和她的 AI 助手。這份文件說明接下來三項功能的需求規格、動工方式、以及注意事項。技術現況請搭配同目錄的 `DEV_TECH_SUMMARY.md`（原開發者整理的 codebase 地圖）一起讀。

## 專案背景

- 遊戲：Word Worm 單字蟲蟲，BookWorm 式網頁拼字遊戲
- 線上版：https://tsun-u.github.io/tsunu-wordworm/
- Repo：https://github.com/Tsun-u/tsunu-wordworm（public，GitHub Pages 部署）
- 現況：v4 Phase 1 已上線——經典模式（7×7 相鄰拼字）＋冒險模式（4×4 全盤任選、回合制戰鬥、一章三隻怪含 Boss）
- 單檔架構：全部邏輯在 `index.html`，零建置、零外部資源

## 這次要做的三項功能

### 1. 戰鬥介面重做（BWA 式版面配置）

把冒險模式的戰鬥畫面改成 Bookworm Adventures 式的配置：

- **上方戰鬥舞台**：左邊蟲蟲、右邊怪物，面對面站立；各自頭上或腳下有血條；玩家目前拼出的字母以「浮字」顯示在兩者中間的舞台上方
- **中下方**：4×4 字母棋盤置中
- **右側欄**：目前怪物的技能一覽（名稱＋小圖示＋簡短說明，例如「鎖定磚：使一格無法選取」）
- **左側欄**：道具欄位置**先做出框架但留空**（顯示「即將推出」之類的佔位），道具系統是下一期的內容
- **下方**：大而明顯的 Attack（攻擊/送出）按鈕，Scramble（清除負面磚）按鈕放旁邊次要位置
- 手機直向時允許重新堆疊（舞台在上、棋盤在中、技能收合為可展開），兩個 RWD 斷點（700px/420px）都要測

注意：現有側邊蟲蟲立繪 `#worm-corner` 與戰鬥舞台裡的蟲蟲會重複，冒險模式下應隱藏側邊版（見 DEV_TECH_SUMMARY 坑 2）。

### 2. 冒險地圖（選關畫面）

- 進入冒險模式時，先顯示一張「冒險地圖」：以路徑串起所有關卡節點，一眼看出總共幾關、目前打到哪
- 已通關：亮色＋勾勾或旗幟；目前可挑戰：高亮；未解鎖：灰色不可點
- 點擊已解鎖的關卡節點即可進入該關（允許回頭重打已通關的關卡）
- 資料結構上目前 `MONSTER_TEMPLATES` 只有一章三隻怪、沒有地圖概念，需要從零設計關卡資料結構（建議：每個節點對應一隻怪或一小段怪物序列，並擴充章節數）
- 進度存檔沿用 `wordworm_adv_progress`，但可能需要擴充欄位（記得同步更新 `isValidAdvState()` 驗證，見 DEV_TECH_SUMMARY 存檔格式一節）

### 3. UI 美術

- 設計稿由小葵（原團隊的美術支援）提供，會另外交付；施工時以設計稿為準
- 設計稿到位前，可先用現有的木質書房風格元素照第 1 項的配置施工，美術後換皮
- 延續零外部資源原則：所有圖形 inline SVG 或 CSS 手刻，不引外部圖片/字型/CDN

## 版權紅線（重要）

- 版面配置參考 Bookworm Adventures（PopCap 商業遊戲）**僅限資訊擺放方式**；美術素材、圖像、音效**一律自製**，不得截取或臨摹原作素材
- 機制設計參考過的 BWA 開源專案（franklindyer，AGPL 授權）：**只能參考玩法概念，程式碼一行都不能抄**——本專案是 MIT-friendly 的 clean-room 實作，抄了 AGPL 程式碼會讓整個專案被授權傳染
- 名稱維持 Word Worm，避開 BookWorm 商標

## 給 AI 的起手 prompt（複製貼上用）

> 你要接手一個網頁拼字遊戲的開發。請先讀 repo 裡的 `HANDOFF.md` 和 `DEV_TECH_SUMMARY.md`，理解需求規格、codebase 結構和九條注意事項之後，跟我確認你的施工計畫，再開始動工。
>
> 這次的任務是 HANDOFF.md 裡的三項功能，建議順序：先做「冒險地圖」（含關卡資料結構重設計），再做「戰鬥介面重做」，美術設計稿到位後再換皮。
>
> 鐵律：(1) 經典模式的行為和存檔格式一個字都不能動；(2) 只能修改冒險模式相關的函式（`*Adventure`）和 DOM（`#adv-*`）；(3) 不引入任何外部資源；(4) 不抄任何 AGPL 程式碼或商業遊戲素材。每完成一個階段先在本地驗證（`python -m http.server` 起服務，經典＋冒險各跑一輪回歸），確認沒問題再 push。

## 交接步驟（Vickie 看這段）

1. 開一個 GitHub 帳號（如果還沒有），到 https://github.com/Tsun-u/tsunu-wordworm 按右上角 **Fork**，遊戲就複製成妳自己的 repo 了
2. 在妳的 fork 裡開啟 **Settings → Pages → Deploy from branch → main / root**，妳就有自己的遊戲網址（`https://妳的帳號.github.io/tsunu-wordworm/`），之後改版都部署在妳這邊
3. 把妳的 repo 網址＋上面那段「起手 prompt」交給妳的 AI（Claude、ChatGPT、Codex 都行），它就知道怎麼開工了
4. 卡關或不確定的地方，隨時回來問阿宇😄
